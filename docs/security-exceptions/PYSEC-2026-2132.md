# PYSEC-2026-2132 scanner-runtime exception

- Assessed: 2026-07-14
- Review deadline: 2026-08-14
- Status: affected dependency, vulnerable function not reachable in the
  agent-eval scanner execution path
- Scope: `src/agent_eval/scanner_runtime/uv.lock` only

## Advisory and locked package

`PYSEC-2026-2132`, also published as `CVE-2026-7246` and
`GHSA-47fr-3ffg-hgmw`, reports command injection in `click.edit()` in Click
8.3.2 and earlier. The published fix is Click 8.3.3.

The scanner runtime locks Semgrep 1.169.0. Its declared requirement is
`click~=8.1.8`, which excludes Click 8.3.3 and resolves to Click 8.1.8. The
Semgrep 1.169.0 source distribution recorded in the lock has this identity:

```text
semgrep-1.169.0.tar.gz
sha256:46932f875b8dff4cb731cd4c908443a0f2f585edbb0a5baa92c4fc033246fdea
```

PyPI was checked on 2026-07-14. Semgrep 1.169.0 was the latest available
release and retained the incompatible Click constraint. Releases 1.122.0
through 1.169.0 use `click~=8.1.8`. Semgrep 1.121.0 nominally permits Click
8.3.3 through `click~=8.1`, but a clean Python 3.12 command smoke test failed
before Semgrep could start, and it is not a safe downgrade from the current
scanner.

## Reachability evidence

The exact Semgrep source distribution above was downloaded, its SHA-256 was
verified, and its extracted source was searched for both direct calls and
imports using these patterns:

```text
click\.edit
from click import[^\n]*\bedit\b
```

Neither pattern occurred. The harness invokes one fixed, noninteractive
command with the absolute path of its packaged `semgrep.yml` ruleset:
`semgrep scan --config <packaged-rules> --no-rewrite-rule-ids
--disable-version-check --metrics off --json --quiet <workspace>`. Callers
cannot select another Semgrep command, ruleset, or append arguments.

Scanner subprocesses receive an allowlisted environment. `EDITOR` and `VISUAL`
are not passed, and Semgrep receives private HOME, XDG configuration, cache,
state, and temporary directories. These controls remove the advisory's editor
command input and the vulnerable function is absent from the locked Semgrep
source path.

## CI handling and removal criteria

The scanner-runtime audit ignores only `PYSEC-2026-2132`. Every other locked
scanner dependency remains subject to the blocking `pip-audit` result. CI also
verifies the exact Semgrep version, Click version, source-distribution hash,
and review deadline before applying the exception. This document does not
claim that the lock has no published advisories.

Remove the exception immediately if any of these occurs:

- Semgrep permits Click 8.3.3 or later.
- Semgrep adds a `click.edit()` call or import.
- The scanner command becomes caller-extensible or interactive.
- `EDITOR`, `VISUAL`, host HOME, or host configuration is passed to scanners.
- The advisory's affected behavior or prerequisites change.

If none occurs first, the exception expires after 2026-08-14. Renewal requires
a new dependency audit, exact-source search, runtime smoke test, and dated
review record.
